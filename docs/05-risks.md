# リスク・注意事項

## 法的・規制リスク

### 情報処理安全確保支援士（RISS）の名称使用

| 項目 | 内容 |
|------|------|
| 資格者の義務 | 登録の維持・CPDポイント取得（年間40ポイント以上）が必要 |
| 名称使用 | 登録を維持している場合のみ「登録情報セキュリティスペシャリスト」を名乗れる |
| 副業での活用 | 資格をアピールすることは問題なし。ただし守秘義務には注意 |

### 副業・就業規則

| チェック項目 | 対応 |
|------------|------|
| 現職の就業規則 | 副業禁止規定がないか確認する |
| 競業避止義務 | 現職と競合するサービスへの従事は禁止の場合あり |
| 確定申告 | 副業収入が年間20万円を超える場合は確定申告が必要 |

### 守秘義務・情報漏洩

- クライアントから受け取った情報をClaude APIに送信する際は**機密情報の取り扱いに注意**
- 契約書に「情報の取り扱い」条項を必ず入れる
- Claude API の利用規約を確認（Anthropicは入力データを学習に使用しないと明言しているが要確認）

## ビジネスリスク

### プランA（セキュリティ文書）

| リスク | 対策 |
|--------|------|
| 生成文書に誤りが含まれる | 必ず専門家（自分）が最終確認を行う旨を契約書に明記 |
| 責任の所在 | 「文書作成支援」であり法的責任は顧客が負う旨を明記 |
| 価格競争 | RISS資格・実績で差別化し低価格競争に巻き込まれない |

### プランB（コードレビューSaaS）

| リスク | 対策 |
|--------|------|
| Claude API 価格変動 | 使用量に上限を設けて利益率を確保する |
| 競合サービスの参入 | 早期に日本市場でのブランドを確立する |
| 継続的な保守負担 | 最初からシンプルな設計を心がける |

### プランC（技術ブログ）

| リスク | 対策 |
|--------|------|
| 収益化まで時間がかかる | プランAを先行させ、ブログは並行スタート |
| AI生成コンテンツの評価低下 | 実体験・実機検証を必ず加える（差別化） |
| プラットフォームリスク | note・Zenn・自社ブログの3箇所に分散 |

## Claude API コスト試算

### プランA（セキュリティ文書）

| 処理 | トークン数 | 月10件時のコスト |
|------|-----------|----------------|
| ヒアリング解析 | 1,000 tokens | 約10円 |
| ドキュメント生成 | 8,000 tokens | 約80円 |
| 反復修正（3回） | 15,000 tokens | 約150円 |
| **1件あたり合計** | | **約240円** |
| **月10件合計** | | **約2,400円** |

単価5万円の受注に対し、APIコストは約2,400円（**コスト率0.05%**）。

### プランB（SaaS）

| 条件 | APIコスト |
|------|----------|
| PR 1件あたりのコスト | 約50〜200円 |
| 月200PR処理時 | 約1〜4万円 |
| 月額収入（200PR相当） | 約10〜21万円 |

PR数が増えるとAPIコストも増えるため、**利用制限の設計が重要**。
