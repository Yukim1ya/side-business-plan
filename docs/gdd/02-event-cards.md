# GDD: イベントカード設計（20枚）

## 設計方針

- **選択肢は必ず2択**。どちらも一長一短で「正解なし」の設計にする
- 効果は**パラメータ設計書の値を参照**して一貫性を持たせる
- カテゴリは5種類: 🐛技術 / 👔クライアント / 👥チーム / 🌐外部 / ✨ポジティブ
- 発生確率は**技術的負債・モラル・満足度**の値に連動して変動する

---

## 🐛 技術系イベント（5枚）

---

### EVT-001: 本番環境で重大バグ発覚

**発生トリガー**: 品質スコア < 60 のとき確率+30%

> スプリント実行中、クライアントから緊急連絡が入った。  
> 「本番で決済処理がエラーになっている。今すぐ直してくれ」  
> 調査すると、先週リリースした機能に致命的なバグがあった。

| | 選択肢A: 即時対応（全員で修正） | 選択肢B: 並行対応（一部だけ修正班） |
|--|--|--|
| 予算 | -30万円（残業費） | ±0 |
| 残り日数 | -3日（修正・リリース） | -1日 |
| 品質スコア | +10 | +3 |
| チームモラル | -15（突然の残業） | -5 |
| クライアント満足度 | +20（迅速対応） | +5 |
| 技術的負債 | -5（根本対応） | ±0 |
| **総評** | 短期ダメージ大、信頼回復 | 被害を最小化するが根本未解決 |

|--|--|--|
| 予算 | -20万円（対応工数） | ±0 |
| 残り日数 | -4日 | ±0 |
| 進捗 | ±0（通常開発が止まる） | +通常分 |
| 品質スコア | +5 | -5 |

| **特殊効果** | なし | 後のスプリントで「情報漏洩事故」イベントが高確率発生 |

> 大きなマイルストーンをクリアした。クライアントも喜んでいる。  
> この勢いをどう活かすか。

| | 選択肢A: チームランチでお祝い（ちょっとした投資） | 選択肢B: この勢いで次のスプリントも全力で |
|--|--|--|
| 予算 | -3万円（ランチ代） | ±0 |
| チームモラル | +25 | +10 |
| 次スプリントベロシティ | +5（ハイモチベ継続） | +2 |
| 技術的負債 | ±0 | +5（ハイペース継続のリスク） |
| **総評** | 人への小さな投資が大きなリターン | 短期加速を優先 |
