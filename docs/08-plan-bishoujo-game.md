# プランE: 美少女ゲーム開発（インディーVN / ギャルゲー）

## 概要

ビジュアルノベル（VN）またはギャルゲーをインディー作品として制作・販売する。
DLsiteを中心に販売し、**エンジニアとしての技術力向上と収益化を両立**できる数少ないプランの一つ。

**このプランを選ぶ理由**:
- ゲーム開発に必要なスキルセット（Unity / スクリプティング / アセットパイプライン / UI/UX）が一気に習得できる
- 日本のインディーゲーム市場（DLsite）は個人開発者でも月数万〜数十万円稼ぐ事例が多数
- AIイラスト生成（Stable Diffusion）で絵師なしでも制作可能（技術的差別化）

## 技術スタック選択

### Option A: Ren'Py（推奨 — まず動かすなら）

```
言語: Python (Ren'Py Script)
特徴: VN専用エンジン。セーブ/ロード/回想CG/テキスト演出が標準装備
配布: Windows / Mac / Linux / Android / iOS / Web 対応
```

**メリット**: VN機能がビルトインで学習コストが最小。MVPまで1〜2ヶ月。
**デメリット**: エンジニアとしての技術的挑戦は低め。

### Option B: Unity（推奨 — 技術力を磨くなら）

```
言語: C#
UIフレームワーク: Unity UI Toolkit / TextMeshPro
ダイアログ管理: Ink (Inkle社のnarrative scripting language)
キャラクターアニメ: Live2D Cubism SDK (または Spine)
オーディオ: FMOD / Wwise
```

**メリット**: 既存のUnityスキルが活きる。Live2D統合・シェーダー演出など技術的挑戦が豊富。
**デメリット**: VN機能を自作する必要があり、完成まで3〜4ヶ月。

> **結論**: **Phase 1 → Ren'Py**（早期リリース・市場検証）、**Phase 2 → Unity**（本格作品）

## AIイラスト生成パイプライン（技術的コア）

絵師を雇う費用（キャラクター1枚 1〜5万円）をなくし、エンジニアスキルで代替する。

```
① キャラクターデザイン (Stable Diffusion SDXL / NovelAI)
   → LoRAで固定キャラを一貫生成
   → 表情差分（通常・笑顔・怒り・照れ・泣き）を自動バリエーション生成

② 背景生成 (Stable Diffusion)
   → 学校・自室・屋上・夕暮れ等のシチュエーション別に生成

③ CG（一枚絵）生成
   → img2img + ControlNet でポーズ指定した一枚絵を制作

④ UI/ロゴ素材
   → タイトル画面・ボタン等はFigmaで制作
```

**ツール**:
- [Stable Diffusion WebUI (AUTOMATIC1111)](https://github.com/AUTOMATIC1111/stable-diffusion-webui) — ローカル実行（RTX 3060 12GB 対応）
- 推奨モデル: Illustrious XL / Pony Diffusion / CivitAI の人気アニメ系モデル
- LoRA学習: Kohya_ss でオリジナルキャラのLoRAを作成

> RTX 3060 12GB があれば全てローカルで完結 — APIコスト不要。

## ゲーム設計（MVP版）

### 規模感（Phase 1 MVP）

| 要素 | 目標値 |
|------|--------|
| ヒロイン数 | 1〜2人 |
| シナリオ文字数 | 3〜5万文字（プレイ時間2〜3時間） |

```
ジャンル: 日常恋愛（難易度低・需要高）
舞台: 高校 or 大学（定番・共感しやすい）
テーマ: 技術者視点で「プログラミング部」「AIが主人公のパートナー」等も差別化になる
```

## 開発ロードマップ

### Phase 1: プロトタイプ（4〜6週間）

- [ ] Ren'Pyセットアップ・チュートリアル完走
- [ ] Stable Diffusionでメインキャラのデザイン確定（立ち絵5差分）
- [ ] LoRA作成 → 一貫した外見を複数シーンで生成できる状態に
- [ ] シナリオ第1話（1万文字）の執筆・実装
- [ ] BGM/SE の調達（魔王魂 / DOVA-SYNDROME 等の無料素材）

**成功基準**: 友人や同人イベントで見せて「面白そう」と言われる

### Phase 2: 完成版制作（2〜3ヶ月）

- [ ] シナリオ全話完成（3〜5万文字）
- [ ] CG5〜10枚制作
- [ ] セーブ/ロード・コンフィグ・CGギャラリー画面の整備
- [ ] タイトル画面・UI 制作（Figma → 素材書き出し）
- [ ] ボイス対応検討（VOICEVOX等の音声合成で代替可）
- [ ] DLsite同人への出品申請

**成功基準**: DLsiteでDL開始・レビュー1件以上

### Phase 3: 拡張・マーケティング（3〜6ヶ月）

- [ ] Unity版での作り直し（Live2D × システム自作でスキル深堀り）
- [ ] X（Twitter）でキャラ紹介・制作日誌を定期投稿 → フォロワー獲得
- [ ] Steam Greenlightへ申請（グローバル展開）
- [ ] Ci-en でサポーター募集（月額支援）

## 競合・差別化

| 競合 | 特徴 | 差別化ポイント |
|------|------|--------------|
| 大手ゲームメーカー | 予算大・品質高 | 個人ならではのニッチ設定・濃いストーリー |
| 同人VN作品 | 数が多い | AIイラスト × エンジニア視点の演出技術 |
| 海外インディーVN | 英語圏向け | 日本語ローカライズ品質の高さ |

> **AI生成アートをエンジニアが使う**こと自体が現時点では差別化になる。同人界でもまだ少数派。

## リスクと対策

| リスク | 対策 |
|--------|------|
| シナリオが完成しない | 文字数制限を設ける（MVP=3万文字上限） |
| AIイラストの品質が安定しない | Phase 1はLoRA作成に時間を集中投資 |
| 販売数が伸びない | X投稿・同人即売会でファン基盤を先に作る |
| 著作権（AIアート） | 学習済みモデルの利用規約を事前確認。商用利用OKのモデルのみ使用 |
