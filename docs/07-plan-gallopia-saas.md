# プランD: 競馬AI予測サブスクリプションSaaS（GALLOPIA商用版）

## 概要

自前のLightGBMモデル（AUC=0.8028）を武器に、競馬ファン向けの**AI予測情報サービス**をWebSaaSとして提供する。
既存のGALLOPIA POCをそのまま商用化できるため、他のどのプランよりも**技術的モートが高い**。

**最大の差別化ポイント**: 独自MLモデル（55特徴量・LightGBM）× リアルタイムWebUI × 月額サブスク

> **法的位置づけ**: 競馬予想情報の提供（情報提供業）は日本で合法。JRAレース結果に基づくAI分析情報の提供であり、馬券購入を代行・斡旋するものではない。

## 技術アーキテクチャ

```
[ データ取得層 ]
  netkipa スクレイピング / JRA-VAN API（将来）
          ↓
[ ML推論層（既存資産）]
  LightGBM（55特徴量） → 予測スコア + EV計算
          ↓
[ APIサーバー ]
  FastAPI + PostgreSQL（Supabase）
  - /races/{race_id}/predictions
  - /subscription (Stripe Webhookで課金状態管理)
          ↓
[ フロントエンド ]
  Next.js + Tailwind CSS
  - レース予測ダッシュボード
  - バックテスト可視化（Recharts）
  - ユーザー管理（Supabase Auth）
          ↓
[ 通知層（プロプラン）]
  LINE Messaging API / Slack Webhook
          ↓
[ インフラ ]
  VPS（ConoHa WING 月1,000円〜）+ GitHub Actions CI/CD
```

## リスクと対策

| リスク | 対策 |
|--------|------|
| モデル精度の一時低下 | バックテスト結果を透明公開し「予測精度の波」をユーザーに説明 |
| データ取得規約違反 | JRA-VAN（公式API）へ移行。スクレイピングはあくまで開発初期のみ |
| 解約率（チャーン）が高い | 月1の的中実績レポートで継続価値を訴求 |
| 競合の参入 | 55特徴量モデルの継続改良で技術的モートを維持 |

---

## 法的注意事項

- **情報提供業として運営**: 「競馬予測情報の提供」であり、馬券の代理購入・賭けの斡旋は行わない
- **免責事項の明記**: 「当サービスの情報は投資・投機を推奨するものではない」
- **景品表示法**: 「必ず当たる」等の誇大広告は禁止。実績データを正直に公開
- **個人情報保護**: Stripeは個人情報不保持（カード情報非保持化対応済み）

## 今すぐできるネクストアクション

```bash
# 1. GALLOPIA POCのモデルをAPIとして動かす
cd /mnt/c/tools/gallopia-poc
pip install fastapi uvicorn

# 2. 最小FastAPIサーバーを作る
cat > api/main.py << 'EOF'
from fastapi import FastAPI
from predict import predict_race  # 既存の予測関数

app = FastAPI()

@app.get("/races/{race_id}/predictions")
async def get_predictions(race_id: str):
    return predict_race(race_id)
EOF

uvicorn api.main:app --reload

# 3. Next.jsフロントエンドを初期化
npx create-next-app@latest gallopia-web --typescript --tailwind
```
